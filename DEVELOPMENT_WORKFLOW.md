# 🚀 개발 워크플로우 규칙

## ⚠️ 중요: 빌드/배포 규칙

### 🚫 절대 하지 말아야 할 것
- **기능 개발 중간에 빌드/배포 금지**
- **테스트 없이 배포 금지**
- **사용자 확인 없이 배포 금지**

### ✅ 올바른 개발 순서
1. **기능 개발** → 완전히 끝날 때까지
2. **로컬 테스트** → 모든 기능 정상 작동 확인
3. **사용자 확인** → "기능 개발 완료, 빌드/배포 진행해도 될까요?"
4. **빌드** → `npm run build`
5. **배포** → `./scripts/deploy-dev.sh`

## 📋 체크리스트

### 기능 개발 단계
- [ ] 요구사항 분석 완료
- [ ] 데이터베이스 스키마 설계/수정
- [ ] 더미 데이터 준비
- [ ] 컴포넌트/훅 개발
- [ ] UI/UX 구현
- [ ] 에러 처리 구현

### 테스트 단계
- [ ] 로컬에서 기능 테스트
- [ ] 데이터베이스 연결 확인
- [ ] UI 렌더링 확인
- [ ] 에러 상황 테스트

### 배포 전 확인
- [ ] 사용자에게 "기능 개발 완료, 빌드/배포 진행해도 될까요?" 확인
- [ ] 사용자 승인 후 빌드/배포 진행

## 💡 기억하기 쉬운 문구
